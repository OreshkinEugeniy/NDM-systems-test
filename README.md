# Тестовое задание на позицию DevOps NDM Systems `Орешкин Евгений`

В папке "app" находится приложение для тестового стенда и Dockerfile lя сборки образа для этого приложения, к которому обращается клиент.
в папке "nginx_confs" находятся конфиги для 3 серверов nginx, играющих роль последовательных proxy_pass.
В файле "docker-compose.yml" описана конфигурация всех контейнеров - 3 контейнера nginx и 1 контейнер с приложением (app)

Возможные схемы обращений от клиента к приложению выглядят так:

пользователь>nginx1:81>nginx2:82>nginx3:83>приложение:80

пользователь>nginx2:82>nginx3:83>приложение:80

пользователь>nginx3:83>приложение:80

Порты открыты только для nginx серверов - 81, 82, 83. Напрямую к приложению пролщьователья обратиться не может.

При обращении к любому из nginx серверов от именни недоверенного узла сохраняется новый ip-адрес с помощью директивы X-Forwarded-For = $remote_addr. Если обращение идет от доверенного узла - то директива X-Forwarded-For добавляет в конец списка цепочки тот ip-адрес, от которого запрос ($proxy_add_x_forwarded_for).

Чтобы определить доверенный ли узел, запросы к nginx могут призодить по двум портам. Если порт 80 - то узел недоверенный, тогда цепочка записывает новый (рельный ip отправителя) и даяльше запрос проксируется на следующий в цепочке nginx сервер на порт 8080. Если запрос приходит на порт 8080 - то узел отправики считается доверенным и тогда ip адрес отправителя дописывается в конец цепочки.

Чтобы запустить стенд необходимо запустить контейнеры:
- при первом запуске, чтобы собрать образ:
  
```
docker compose up --build -d
```

- и при повторных запусках:

```
docker compose up -d
```

Для проверки цепочки обращаемся к nginx серверам: 

```
curl localhost:81
curl localhost:82
curl localhost:83
```

При каждом обращении увидим свою цепочку.

При попытке передать ложную цепочку ip адресов, все равно будет выведена цепочка адресов с реальным ip отправителя:

```
curl -H "X-Forwarded-For: 1.1.1.1, 2.2.2.2" localhost:81
```
